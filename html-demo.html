Document Structure Core

Every HTML5 document starts with <!DOCTYPE html> and contains <html>, <head>, and <body> elements.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Page Title</title> <meta name="description" content="A description"> <link rel="stylesheet" href="styles.css"> </head> <body> <!-- Visible content --> </body> </html>

Metadata Elements Head

Metadata lives in the <head> and provides information about the document. View the source code of this page to see metadata in action, including charset, viewport, description, keywords, and author meta tags.

<meta charset="UTF-8"> <meta name="viewport" content="width=device-width"> <meta name="description" content="Page description"> <meta name="keywords" content="HTML, CSS, JS"> <meta name="author" content="Developer"> <title>Page Title</title> <link rel="stylesheet" href="style.css"> <style>/* Internal CSS */</style> <script src="app.js"></script>

What is HTML5?

HTML5 is the latest version of HyperText Markup Language, the standard code used to structure web pages. It introduces semantic elements, multimedia support, and APIs for building rich web applications.

Why use semantic elements?

Semantic elements like <article>, <section>, and <nav> give meaning to your content. They improve accessibility for screen readers, enhance SEO, and make your code more readable and maintainable.

What are data attributes?

Data attributes (data-*) allow you to store custom data on HTML elements. They can be accessed via JavaScript using the dataset property, making them perfect for storing state or configuration without extra classes.

<details> <summary>Click to expand</summary> <p>Hidden content revealed</p> </details> <!-- Open by default --> <details open> <summary>Already open</summary> <p>This content is visible</p> </details>
